# PHP — Composer

Composer is the PHP dependency manager. It handles package installation,
autoloading, and script execution.

## composer.json Structure

```json
{
    "name": "my-org/my-project",
    "description": "A PHP project",
    "type": "project",
    "require": {
        "php": ">=8.2",
        "laravel/framework": "^11.0",
        "guzzlehttp/guzzle": "^7.0"
    },
    "require-dev": {
        "phpunit/phpunit": "^11.0",
        "laravel/pint": "^1.0"
    },
    "autoload": {
        "psr-4": {
            "App\\": "src/"
        }
    },
    "autoload-dev": {
        "psr-4": {
            "Tests\\": "tests/"
        }
    },
    "scripts": {
        "test": "phpunit",
        "lint": "pint",
        "fix": "pint --fix"
    },
    "config": {
        "optimize-autoloader": true,
        "sort-packages": true
    }
}
```

## require vs require-dev

| Key | Purpose |
|---|---|
| `require` | Production dependencies shipped with the app |
| `require-dev` | Development-only (testing, linting, static analysis) |

```bash
composer install                  # installs both
composer install --no-dev         # production only
composer require guzzlehttp/guzzle:^7.0
composer require --dev phpunit/phpunit:^11.0
composer remove vendor/package
```

## Autoloading (PSR-4)

Maps namespaces to directory structures. PSR-4 is the standard.

```
src/
├── Models/
│   └── User.php          → App\Models\User
├── Services/
│   └── Payment.php       → App\Services\Payment
└── Helpers.php           → App\Helpers
```

```json
"autoload": {
    "psr-4": {
        "App\\": "src/"
    }
}
```

After adding autoload entries, regenerate the autoloader:

```bash
composer dump-autoload
```

## Classmap & Files Autoload

```json
"autoload": {
    "classmap": [
        "legacy/",
        "lib/old-lib.php"
    ],
    "files": [
        "src/helpers.php",
        "src/constants.php"
    ]
}
```

- **classmap**: Scans directories for classes — used for non-PSR-4 code.
- **files**: Always included on every request — ideal for helper functions.

## composer.lock

Generated by `composer install` / `composer update`. Commits exact versions
enabling reproducible builds.

```bash
composer install     # install from lock file (deterministic)
composer update      # update to latest allowed versions, rewrite lock
composer update guzzlehttp/guzzle  # update a single package
```

## vendor/bin

Installed binaries (PHPUnit, Pint, PHPStan) live in `vendor/bin/`.

```bash
./vendor/bin/phpunit
./vendor/bin/pint

# Or via composer scripts
composer test
composer lint
```

## Scripts

Run custom commands during Composer lifecycle events.

```json
"scripts": {
    "post-install-cmd": [
        "php artisan optimize"
    ],
    "post-update-cmd": [
        "php artisan optimize"
    ]
}
```

Lifecycle hooks: `pre-install-cmd`, `post-install-cmd`, `pre-update-cmd`,
`post-update-cmd`, `post-autoload-dump`.
